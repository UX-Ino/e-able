# KWCAG 2.1 vs 2.2 비교 가이드

본 문서는 한국형 웹 콘텐츠 접근성 지침(KWCAG) 2.1과 2.2(KS X 3001:2024)의 주요 차이점과 검사 항목을 비교하여 정리한 가이드입니다.

## 1. 개요 비교

| 구분 | KWCAG 2.1 (KS X OT0003:2015) | KWCAG 2.2 (KS X 3001-1:2024) |
|:---|:---|:---|
| **검사 항목 수** | 24개 | **33개 (+9개)** |
| **원칙** | 4개 원칙 (동일) | 4개 원칙 (동일) |
| **지침** | 13개 지침 | **14개 지침 (+1개)** |
| **주요 특징** | 웹 환경 중심의 기본 접근성 | 모바일 접근성, 인지 장애, 저시력자 배려 강화 |

---

## 2. 신규 추가 항목 (9개)

KWCAG 2.2에서 새롭게 추가된 9개 항목은 국제 표준인 WCAG 2.1/2.2의 핵심 항목들을 국내 환경에 맞춰 도입한 것입니다.

| 번호 | 검사 항목명 | 지침 번호 | 주요 내용 |
|:---:|:---|:---:|:---|
| 1 | **문자 단축키** | 2.1.4 | 단일 문자만으로 작동하는 단축키 사용 시 오작동 방지 수단 제공 (해제 또는 변경) |
| 2 | **고정된 참조 위치 정보** | 2.4.7 | 전자출판문서 등에서 페이지 이동 시 참조 위치를 일관되게 제공 |
| 3 | **단일 포인터 입력 지원** | 2.5.1 | 다중 터치나 복잡한 동작 대신 단일 포인터(클릭, 터치)로도 조작 가능해야 함 |
| 4 | **레이블과 네임** | 3.2.4 | 시각적 레이블 텍스트와 보이지 않는 코딩상의 이름(Name)을 일치시켜 음성 인식 오류 방지 |
| 5 | **동작 기반 작동** | 3.2.5 | 기기 흔들기 등 동작 기반 기능을 일반 UI 컨트롤로도 조작 가능하게 제공 |
| 6 | **포인터 입력 취소** | 3.2.6 | 터치/클릭 시 누를 때가 아닌 뗄 때 실행되게 하거나, 실행 전 취소 수단 제공 |
| 7 | **찾기 쉬운 도움 정보** | 3.4.3 | 도움말 정보(연락처, 챗봇 등)를 사이트 전체에서 일관된 위치에 배치 |
| 8 | **접근 가능한 인증** | 3.4.4 | 퍼즐 맞추기, 암기 등 인지적 테스트가 필요한 인증 시 대체 수단 제공 |
| 9 | **반복 입력 정보** | 3.4.5 | 동일한 프로세스 내에서 이전에 입력한 정보를 다시 입력하지 않도록 자동 완성 등 제공 |

---

## 3. 전체 항목 비교 상세

KWCAG 2.1의 기본 24개 항목은 2.2에서도 그대로 유지되거나 일부 용어가 명확화되었습니다.

### 원칙 1. 인식의 용이성 (Perceivable)
1. **적절한 대체 텍스트 제공** (1.1.1)
2. **자막 제공** (1.2.1)
3. **수어 제공** (선택 항목)
4. **표의 구성** (1.3.1)
5. **콘텐츠의 선형 구조** (1.3.2)
6. **명확한 지시사항 제공** (1.3.3)
7. **색에 무관한 콘텐츠 인식** (1.4.1)
8. **자동 재생 금지** (1.4.2)
9. **텍스트 콘텐츠의 명도 대비** (1.4.3)
10. **콘텐츠 간의 구분** (1.4.4)

### 원칙 2. 운용의 용이성 (Operable)
11. **키보드 사용 보장** (2.1.1)
12. **초점 이동과 표시** (2.1.2)
13. **조작 가능** (2.1.3)
14. ✨ **문자 단축키** (2.1.4 - **New**)
15. **응답 시간 조절** (2.2.1)
16. **정지 기능 제공** (2.2.2)
17. **깜빡임과 번쩍임 사용 제한** (2.3.1)
18. **반복 영역 건너뛰기** (2.4.1)
19. **제목 제공** (2.4.2)
20. **적절한 링크 텍스트** (2.4.3)
21. ✨ **고정된 참조 위치 정보** (2.4.7 - **New**)
22. ✨ **단일 포인터 입력 지원** (2.5.1 - **New**)

### 원칙 3. 이해의 용이성 (Understandable)
23. **기본 언어 표시** (3.1.1)
24. **사용자 요구에 따른 실행** (3.2.1)
25. ✨ **레이블과 네임** (3.2.4 - **New**)
26. ✨ **동작 기반 작동** (3.2.5 - **New**)
27. ✨ **포인터 입력 취소** (3.2.6 - **New**)
28. **오류 정정** (3.3.1)
29. **레이블 제공** (3.4.1)
30. ✨ **찾기 쉬운 도움 정보** (3.4.3 - **New**)
31. ✨ **접근 가능한 인증** (3.4.4 - **New**)
32. ✨ **반복 입력 정보** (3.4.5 - **New**)

### 원칙 4. 견고성 (Robust)
33. **마크업 오류 방지** (4.1.1)
34. **웹 애플리케이션 접근성 준수** (4.2.1)

> [!NOTE]
> KWCAG 2.2는 최신 기술(모바일, 터치 인터페이스)에 대한 대응력을 높였으므로, 신규 프로젝트 진행 시 2.2 기준을 따르는 것을 권장합니다.
